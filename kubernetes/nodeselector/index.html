    nodeName和nodeSelector¶

    nodeName¶

    Pod.spec.nodeName 用于强制约束将 Pod 调度到指定的 Node 节点上，这里说是“调度”，但其实指定了 nodeName 的Pod会直接跳过 Scheduler 的调度逻辑，直接写入 PodList 列表，该匹配规则是 强制匹配 。

    apiVersion: extensions/v1beta1
kind: Deployment
metadata:
  name: tomcat
spec:
  replicas: 1
  template:
    metadata:
      labels:
        app: tomcat-app
    spec:
      nodeName: m01
      containers:
      - name: tomcat
        image: tomcat
        ports:
        - containerPort: 8080

    nodeSelector¶

    Pod.spec.nodeSelector 是通过kubernetes的 label-selector 机制进行节点选择，由scheduler调度策略 MatchNodeSelector 进行 label匹配，调度pod到目标节点，该匹配规则是强制约束。启用节点选择器的步骤为

    1. Node添加label¶
    kubectl label nodes <node-name> <label-key>=<label-value>
    2. 确认label¶
    kubectl get nodes <node-name> --show-labels
    3. 调度Deployment¶
    apiVersion: extensions/v1beta1
kind: Deployment
metadata:
  name: tomcat
spec:
  replicas: 1
  template:
    metadata:
      labels:
        app: tomcat-app
    spec:
      nodeSelector:
        tomcat/role: dev
      containers:
      - name: tomcat
        image: tomcat
        ports:
        - containerPort: 8080
